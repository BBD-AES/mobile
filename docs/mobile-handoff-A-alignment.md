# 핸드오프(design_handoff_A) 정렬 요청 — 코드/결정 기준 (2026-06-25)

> 대상: 디자이너(고충실도 핸드오프 `mobile-design-handoff-A.md` 작성자).
> 배경: 핸드오프 정적 문서가 작성 시점(06-24) **구(舊) 상태**를 기술 중이고, Compose 구현 + 리뷰 B 결정이 이미 **신(新) 상태**로 이동함. **코드가 권위**이며 핸드오프를 아래에 맞춰 갱신 요청.
> 용어: SO=**재고이동요청** · IN_FULFILLMENT=**도착 대기** · RECEIVED=**입고 완료**. 상세는 [mobile-review-B.md](mobile-review-B.md).

가장 큰 갭: §3.6 작업이력(개인 received_by) → 지점 전체 이동요청 이력 / §3.5 부품상세 '최근 입고' → '도착 예정' / §3.1·§2 홈 그라데이션 히어로 → 플랫 카드 / §7 Radius 4·6px → 실제 13·16dp.

---

## P1 — 부품 상세: "최근 입고" → "도착 예정" (§3.5)
- **핸드오프**: 부품 상세 시트에 과거 RECEIVED "최근 입고 라인" 표시.
- **코드/결정**: 과거 입고가 아니라 **이 SKU가 든 IN_FULFILLMENT(이동 중) 발주 2~3건** = "도착 예정"(출처 창고·곧 들어올 수량, 중립 "이동 중"만, **ETA/지연 표기 금지**). 과거 입고완료는 이동요청 이력 탭으로.
- **요청**: §3.5 "최근 입고 라인" → "도착 예정 섹션(IN_FULFILLMENT 2~3건 · 출처 창고 · 입고 예정 수량 · '이동 중' 라벨, ETA 금지)". "과거 입고 이력은 이동요청 이력 탭" 명시.

## P2 — 부품 상세 1차 액션 = "이 품목 출고" 1개 (§3.5/§3.2)
- **핸드오프**: SKU 단위 입고가 없다는 점 미명시.
- **코드/결정**: 부품 상세 유일 액션 = **"이 품목 출고"**(SKU 프리필 → 출고 스캔). 입고 스캔/STR(웹) 버튼 없음. SKU 단위 입고는 모델상 부재(입고=SO 전량).
- **요청**: §3.5에 '1차 액션 = "이 품목 출고" 1개뿐. SKU 단위 입고 버튼 금지(입고는 SO 전량 모델)' 명시.

## P3 — 작업 이력 → "이동요청 이력"(지점 전체) (§3.6/§2/§4.2)
- **핸드오프**: §3.6 "작업 이력" · 목적 "내가 입고 확인한 발주 이력" · API `received_by={내 사번}`. §2 탭 라벨 "작업이력".
- **코드/결정**: 화면명 **"이동요청 이력"**, 지점 전체 STR을 **상태 무관** 표시, 쿼리 `to_warehouse_code`(received_by 제거, 서버가 지점 자동 스코프), **요청자/입고자/취소자 + 일시**, 상태 필터 칩(전체/도착 대기/백오더/입고 완료/취소·반려) + 검색, 진행 중(도착대기·백오더) 상단 정렬.
- **요청**: §3.6 제목/목적/API/구성 전면 갱신, §2 탭 라벨 "작업이력"→"이동요청 이력", §4.2 쿼리 라인 갱신.

## P4 — 홈: 그라데이션 히어로 제거 + "최근 입고 완료" (§3.1/§2)
- **핸드오프**: §3.1 "히어로: 도착 대기 N건"(강조 트리트먼트), "최근 입고 5건".
- **코드/결정**: 파란 그라데이션 hero 제거 → **플랫 업무 카드**(숫자만 accent, 그림자/CTA 알약 없음). 섹션명 **"최근 입고 완료"**(미리보기 3건, 더보기→이동요청 이력).
- **요청**: §3.1 '도착 대기 카드(그라데이션/hero 금지, 플랫·숫자만 accent)', §2 "홈 히어로에서 진입"→"홈 도착 대기 카드에서 진입", "최근 입고 5건"→"최근 입고 완료(미리보기 3건)".

## P5 — 재고 필터: 상태/카테고리 2단 분리 (§3.5)
- **핸드오프**: "카테고리 칩 + 안전재고 미달 토글 + 검색".
- **코드/결정**: 상태(전체/정상/부족/없음)·카테고리(전체/분류)를 **독립 2단 칩 줄**, 각 줄 단일선택, **AND 결합**. 별도 미달 토글 없음(상태 칩 '부족'/'없음'이 대체).
- **요청**: §3.5 필터를 '2단 칩(1행 상태 / 2행 카테고리), 각 단일선택, AND'로 교체. '미달 토글' 표현 삭제.

## P6 — 마이: 제거 항목 (§3.7)
- **핸드오프**: §3.7 "…/ 이용 권한 / 앱 정보 / 로그아웃" + 상태 "지점 매핑 대기 안내".
- **코드/결정**: 마이에서 ①매핑대기 ②내 작업이력 링크 ③지점 알림함 준비중 ④앱 정보 v0.6 모두 제거. 잔존 = 프로필 · 이용 권한 · 비밀번호(Keycloak) · 로그아웃.
- **요청**: §3.7 구성 '프로필 / 이용 권한 / 비밀번호(Keycloak) / 로그아웃'. '지점 매핑 대기'는 마이에서 제거(재고 화면의 tenancy 폴백 안내만 유지). '내 작업이력 링크 없음', '지점 알림함 준비중 없음' 명시.

## P7 — 용어 통일: SO·IN_FULFILLMENT·발주 (§3.2·§3.6·§4·§7)
- **핸드오프**: 받는 SO를 "발주/발주번호/발주 품목"으로 표기. §7 용어표에 SO 정의·IN_FULFILLMENT 라벨 없음.
- **코드/결정**: SO=재고이동요청 · IN_FULFILLMENT=도착 대기("입고 대기" 금지) · RECEIVED=입고 완료. Scan의 '발주'는 오칭 → '요청번호/재고이동요청/입고'로 스윕 완료(이 레포 반영됨).
- **요청**: §7에 위 3개 정의 행 추가. §3.2 입고 스캔 흐름의 '발주/발주번호/발주 품목'을 '입고 QR/요청번호/입고 품목'으로 치환(코드와 일치).

## P8 — 디자인 토큰: Radius (§7)
- **핸드오프**: "Radius 4px(카드/버튼) · 6px(모달/시트)".
- **코드/결정**: bbdCard 13dp · 칩/필드/버튼 13~14dp · 모달 16dp · 시트 상단 22dp(둥근 모서리). 4/6px는 실제 값과 불일치.
- **요청**: §7 Radius 행을 실제 값으로 갱신 + '과한 radius/그림자 금지(카드 그림자 약하게)' 주석 추가.

## P9 — anti-AI 티: 그라데이션/hero/장식 문구 회피 (전역)
- **핸드오프**: §2·§3.1 '히어로', §3.3 화려한 배너 등 강조 트리트먼트 전제. 회피 원칙 미명문화.
- **코드/결정**: 홈 그라데이션 hero 제거→플랫 카드, 공용 카드 shadow 3→2·radius 16→13, 모달 20/24→16/16. 실제 운영자용 업무 화면 톤(장식 최소, 숫자/상태 위계).
- **요청**: 핸드오프에 'anti-AI/밀도 원칙' 소절 신설 — ①그라데이션·hero·과한 그림자/radius 금지 ②장식 마케팅 문구 금지(현장 동사형 짧은 라벨) ③카드 shadow 2단계·모달 16dp·radius 13dp 기준. §2·§3.1·§3.3 '히어로'→'카드' 치환.
