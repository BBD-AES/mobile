## 사실 확인 (질문 답)
- **매핑대기**: 백엔드 상태 아님. user `/users/me`는 `tenancyName`(지점/창고명)만, `warehouseCode` 없음 → inventory 창고목록 이름매칭(PR#18)으로 해결. → 마이의 "매핑대기/연동 준비중"은 가상 상태라 **제거**.
- **알림**: `Notification.targetRole` 버킷(HQ_MANAGER / 지점 창고명)뿐. **개인 알림 없음** = 지점 공유 1종. `GET /notifications`가 지점 호출자에 지점 알림 반환(PR#84). → "내 알림 vs 지점 알림" 구분 폐기, "준비중"도 실제론 연동 가능.
- **용어**: IN_FULFILLMENT="도착 대기"(모바일), "처리중(충당)"(웹). **"입고 대기"는 오용** → 도착 대기로.
- **지점 이력 쿼리**: `GET /api/v1/sales-orders` 지점스코프 + status·requested_by·received_by·기간 + 응답에 requestedBy/approvedBy/receivedBy/canceledBy 모두 있음.
